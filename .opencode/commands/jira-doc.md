Generate a complete Jira technical-functional documentation in Markdown.

Requirements:
- Include summary, objective, context, and scope.
- Include a complete flow explanation.
- Include a Mermaid diagram.
- Describe implemented functionalities by section in detail, focused on behavior rather than code.
- Include tables for files, datasets, tables, columns, or resources used.
- Include input and output examples.
- If there are multiple execution modes, explain each one.
- Include pending items if any.
- Include related references or links if relevant.
- End with executive summary and acceptance criteria checklist.

Expected structure:
1. Summary
2. Objective
3. Context
4. Scope
5. Complete flow
6. Mermaid diagram
7. Implemented functionalities
8. Data/resources used
9. Input/Output
10. Execution options
11. Representative cases
12. Pending items
13. References/attachments
14. Executive summary
15. Acceptance criteria checklist

Use the current repository context and changed files.
If information is missing, state assumptions clearly.
