# Architecture Documentation Workflow

## Objetivo
Documentar arquitectura de forma útil, práctica y mantenible.

## Flujo recomendado
1. Definir sistema en foco.
2. Identificar actores y sistemas externos.
3. Elaborar vista de contexto.
4. Identificar contenedores principales.
5. Documentar componentes relevantes si hace falta.
6. Documentar flujos runtime clave.
7. Documentar despliegue si es relevante.
8. Registrar decisiones arquitectónicas.
9. Documentar riesgos, trade-offs y supuestos.
