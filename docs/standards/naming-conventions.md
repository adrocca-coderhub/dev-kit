# Naming Conventions

## Objetivo
Mantener consistencia en nombres de archivos, carpetas y artefactos del kit.

## Reglas recomendadas

### Markdown
Usar `kebab-case`.
Ejemplos:
- `deep-analysis-skill.md`
- `architecture-doc-prompt.md`

### Python
Usar `snake_case`.
Ejemplos:
- `pipeline_etl.py`
- `validate_schema.py`

### Carpetas
Usar nombres cortos y claros en minúsculas.
Ejemplos:
- `prompts`
- `templates`
- `scripts`

### Snippets
Mantener nombres descriptivos y prefijos cortos.
Ejemplos:
- `prdoc`
- `jiradoc`
- `pyetl`

## Recomendación
Evitar mezclar demasiados estilos si el contenido pertenece a la misma categoría.
