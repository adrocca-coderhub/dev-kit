# Diagram Guidelines

## Objetivo
Mantener diagramas claros, consistentes y útiles.

## Reglas generales
- diagramar solo lo necesario
- evitar diagramas gigantes
- usar nombres claros y cortos
- agregar contexto en texto además del diagrama
- mantener consistencia entre nodos y secciones

## Cuándo usar cada tipo
- `flowchart`: para flujos funcionales o ETL
- `sequenceDiagram`: para interacciones paso a paso
- `graph LR/TD`: para vistas simples de arquitectura
- C4-style simplificado: para arquitectura

## Recomendaciones
- si un diagrama no aporta claridad, no incluirlo
- acompañar cada diagrama con explicación textual
- separar diagramas por nivel de detalle
