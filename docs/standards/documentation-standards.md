# Documentation Standards

## Objetivo
Asegurar que la documentación del kit y de los entregables sea clara, consistente y útil.

## Principios
- escribir para que alguien nuevo entienda
- explicar el porqué, no solo el qué
- usar tablas cuando mejoren legibilidad
- separar contexto, flujo, datos y riesgos
- incluir ejemplos cuando el tema lo requiera

## Para PRs
- incluir tabla de archivos modificados
- explicar impacto funcional y técnico
- documentar input/output si aplica
- agregar Mermaid cuando el flujo lo amerite
- no dejar secciones vacías

## Para Jira
- incluir contexto y objetivo
- documentar alcance
- incluir criterios de aceptación
- explicar comportamiento con palabras
- incluir datos y recursos usados

## Para arquitectura
- documentar responsabilidades y relaciones
- incluir al menos contexto y contenedores
- separar arquitectura actual vs propuesta si aplica
- registrar decisiones importantes con ADR cuando corresponda
