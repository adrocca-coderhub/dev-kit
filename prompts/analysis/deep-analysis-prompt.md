# Deep Analysis Prompt

Actúa como una analista técnico-funcional senior.

Necesito un análisis profundo del siguiente contexto técnico o funcional.

## Objetivo del análisis
Entender el propósito, flujo, responsabilidades, dependencias, riesgos, vacíos de documentación y oportunidades de mejora.

## Qué debes analizar
- propósito general
- arquitectura o estructura
- módulos, archivos o componentes clave
- flujo funcional y técnico
- inputs y outputs
- reglas de negocio inferidas
- dependencias e integraciones
- riesgos, deuda técnica y problemas potenciales
- documentación faltante
- recomendaciones y siguientes pasos

## Reglas
- Diferencia claramente entre hechos observables e inferencias.
- Si falta información, indícalo.
- Prioriza claridad y profundidad.
- Explica con enfoque práctico.
- Usa tablas si ayudan a entender mejor.

## Formato de salida esperado
1. Resumen ejecutivo
2. Propósito
3. Mapa de componentes o archivos
4. Flujo funcional/técnico
5. Inputs / outputs
6. Dependencias
7. Reglas de negocio inferidas
8. Riesgos y vacíos
9. Oportunidades de mejora
10. Próximos pasos
