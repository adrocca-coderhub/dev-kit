# Jira Documentation Prompt

Actúa como una especialista en documentación técnico-funcional para tarjetas Jira.

Necesito que redactes una tarjeta Jira de alta calidad, clara, estructurada, robusta y útil tanto para equipos técnicos como funcionales.

## Objetivo
Transformar información técnica, funcional o difusa en una documentación Jira bien organizada, con contexto, alcance, flujo, criterios de aceptación y detalle operativo.

## Reglas obligatorias
- La salida debe ser en Markdown.
- Debe incluir resumen, objetivo y contexto.
- Debe incluir flujo completo con diagrama Mermaid.
- Debe dividir las funcionalidades implementadas por secciones, explicando cada una con detalle en palabras.
- Debe incluir tablas de archivos, datos, columnas, tablas o recursos usados.
- Debe incluir ejemplos de input y output.
- Debe explicar cómo ejecutarlo si aplica, y si hay varias maneras, describir cada una.
- Debe incluir pendientes si existen.
- Debe incluir adjuntos o links relevantes si se proporcionan.
- Debe terminar con un resumen ejecutivo y criterios de aceptación en checklist.
- Debe estar redactado para que alguien nuevo entienda la solución sin revisar el código completo.

## Estructura obligatoria
1. Resumen
2. Objetivo
3. Contexto
4. Alcance
5. Flujo completo
6. Diagrama Mermaid
7. Funcionalidades implementadas
8. Datos utilizados
9. Input / Output
10. Formas de ejecución
11. Casos representativos
12. Pendientes
13. Adjuntos / referencias
14. Resumen ejecutivo final
15. Criterios de aceptación
